Streamlining Store Operations

30-45 minutes Intermediate Tutorial

Scope

Applies to Magento Open Source and Adobe Commerce 2.4.x (Admin-only features; no extensions required). This tutorial uses only features available in Magento Open Source 2.4.x. If you use Adobe Commerce, you can also leverage B2B and advanced features (not covered here).

Why This Tutorial Matters

This tutorial gives you proven techniques to reduce order handling time, prevent stockouts, and standardize fulfillment—improving customer satisfaction and lowering cost per order without adding headcount or custom code.

Business Impact: Reduce order handling time, prevent stockouts, and standardize fulfillment to improve customer satisfaction and lower cost per order.

What You'll Achieve: Cut daily order processing time by 20–40% using saved grid views and mass actions, reduce out-of-stock incidents with inventory thresholds and weekly Low Stock reviews, and standardize invoicing and shipment emails to improve customer communication.

Learning Journey Overview

Your Situation

You process 10–500 orders/day, manage a growing catalog, and need faster picking/packing, fewer manual updates, and consistent customer notifications—without adding headcount or custom code.

What You'll Learn

By completing this tutorial, you will:

  • Create and use saved Admin grid views for Orders and Products
  • Perform common mass actions efficiently on orders and products
  • Configure Sales Emails for orders, invoices, and shipments
  • Set inventory thresholds and review Low Stock reports
  • Structure inventory using Sources and Stocks (MSI)
  • Update product attributes in bulk safely
  • Optimize indexers for performance with Update by Schedule

Success Criteria

You'll know you've succeeded when:

  • You have at least two saved views on Sales > Orders and Catalog > Products
  • New orders trigger Order and Shipment emails automatically
  • Low Stock report is empty or reviewed weekly; no unexpected stockouts in the next 30 days
  • Indexers are set to Update by Schedule and all are in Ready status

Time Investment

  • Estimated time: 30–45 minutes for first-time setup
  • Skill level after completion: Confident day-to-day operator with efficient order and inventory workflows
  • Business value unlock: Faster fulfillment, fewer errors, improved customer communications—no extensions required
  • Note: Ongoing usage typically adds ~5–10 minutes per week for reviews and adjustments

  • Export or screenshot key configuration settings as a lightweight backup

Step-by-Step Learning Path

1 Orders & Fulfillment

Create saved Orders grid views

  1. 1. Go to Sales > Orders.
  2. 2. Click Filters, set Status = Pending, set Purchase Date to Today (or as needed).
  3. 3. Click Columns and enable the columns your team needs (e.g., Grand Total, Shipping Address, Shipping Method).
  4. 4. In the Views selector (top right), click Save Current View and name it "New Orders Today".
  5. 5. Repeat to create an "Awaiting Shipment" view (e.g., Status = Processing, no Shipment created).

Use mass actions to batch process safely

  1. 1. Go to Sales > Orders and select your "New Orders Today" view.
  2. 2. Select multiple orders (check the boxes), open Actions and choose safe batch operations like Print Invoices or Print Packingslips.

Tip: If the Invoice action is unavailable for an order, the payment method may auto-capture; open the order and check the Invoices tab.

2 Customer Communications

Configure Sales Emails

  1. 1. Go to Stores > Configuration > Sales > Sales Emails.
  2. 2. Ensure Order, Invoice, Shipment, and Credit Memo emails are Enabled.
  3. 3. Choose appropriate Email Sender identities (configured under Stores > Configuration > General > Store Email Addresses).
  4. 4. Save Config.
  5. 5. Test: Place a small storefront order and confirm that Order and Shipment emails arrive.

3 Inventory & MSI

Set low stock thresholds and review reports

  1. 1. Go to Stores > Configuration > Catalog > Inventory > Product Stock Options.
  2. 2. Set Notify for Quantity Below to a sensible buffer (e.g., 5–10, depending on sales velocity) and Save Config.
  3. 3. Weekly: Go to Reports > Products > Low Stock, review and restock items before they sell out.

Define Sources and Stocks (for multi-location)

  1. 1. Go to Stores > Inventory > Sources and Add New Source for each physical location (enable and fill address/contact details).
  2. 2. Go to Stores > Inventory > Stocks and either:
    • Use the default Stock for single-location, or
    • Create a new Stock, assign your website, and add the relevant Sources.
  3. 3. When fulfilling orders, Magento's Source Selection Algorithm (SSA) will suggest optimal source(s) based on your configuration.

4 Catalog Efficiency

Bulk update product attributes

  1. 1. Go to Catalog > Products.
  2. 2. Filter/select the products to change.
  3. 3. In Actions, choose Update Attributes.
  4. 4. Adjust fields (e.g., Tax Class, Visibility, or attribute values) and click Save.

Create a "Out of Stock" product view

  1. 1. Go to Catalog > Products.
  2. 2. Click Filters, set Stock Status = Out of Stock (and add any other useful filters like Visibility or Website).
  3. 3. Adjust Columns to include Qty, Stock Status, Status, and Visibility.
  4. 4. Save the view as "Needs Restock" to quickly identify products requiring attention.

5 Performance

Optimize indexers for stability and speed

  1. 1. Go to System > Tools > Index Management.
  2. 2. Select all indexers, choose Actions = Set to Update by Schedule, and Submit.
  3. 3. Verify: Confirm all indexers show Ready status after the next cron run. If not, verify the server cron is running.

When Things Don't Go as Expected

Even experienced merchants encounter challenges. Here's how to handle common situations:

Solution

  • Ensure your admin role has permissions for the area; try clearing Admin cache (System > Cache Management > Flush Magento Cache).

Solution

  • The payment method may auto-capture; open the order and check the Invoices tab.

Solution

  • Confirm Stores > Configuration > Sales > Sales Emails are enabled; verify Mail Sending Settings under Stores > Configuration > Advanced > System; confirm cron and SMTP with your host/developer.

Solution

  • SSA not suggesting sources: Ensure multiple Sources are enabled and assigned to the Stock serving your website.
  • Low Stock report empty but items sell out: Raise Notify for Quantity Below or shorten your review cadence.

Solution

  • Go to System > Tools > Index Management > Reindex Data. If issues persist, verify your server cron is running and check for long-running reindex processes.
