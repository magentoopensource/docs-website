GitHub Contributors Widget

Laravel Integration Guide

Production Ready Laravel 12

What Is This?

A production-grade widget that displays the top 5 contributors from a GitHub repository. Built as a standalone PHP application, it needs to be integrated into your Laravel 12 documentation website.

How It Works

3-Tier Caching Strategy

Tier 1
Memory Cache (1 hour)
PHP array, <1ms access time
Tier 2
Database Cache (30 days)
MySQL table, 5-10ms access time
Tier 3
GitHub API
External fallback, 500-1000ms

Minimal API Usage

The widget uses only 0.04% of GitHub's rate limit through aggressive caching and monthly scheduled updates. Data is fetched once per month via Laravel's scheduler.

Integration Steps

1

Create Database Migration

Create a Laravel migration for the contributors tables: 

php artisan make:migration create_github_contributors_tables

Copy the table definitions from contributor-widget/database/schema.sql into the migration's up() method. Convert MySQL DDL to Laravel Schema Builder syntax.

Note: You may want to adjust table names to follow Laravel conventions (e.g., add prefix or modify naming).
2

Create Laravel Service

Create a service class that wraps the widget functionality: 

php artisan make:class Services/GitHubContributorsService

This service will integrate with the existing widget code from contributor-widget/src/Services/. Use dependency injection for Configuration and Database connections.

3

Add Configuration

Add GitHub configuration to config/services.php: 

'github' => [
    'token' => env('GITHUB_API_TOKEN'),
    'repo_owner' => env('GITHUB_REPO_OWNER'),
    'repo_name' => env('GITHUB_REPO_NAME'),
    'api_url' => 'https://api.github.com',
    'timeout' => 30,
    'cache_ttl_days' => 30,
],
4

Update Environment Variables

Add to your .env file: 

GITHUB_API_TOKEN=ghp_your_token_here
GITHUB_REPO_OWNER=mage-os
GITHUB_REPO_NAME=magento2
Important: The GitHub API token will be provided by the Magento Association website. You do not need to create a new token.
5

Create Blade Component

Create a reusable Blade component for rendering the widget: 

php artisan make:component GitHubContributors

The component should fetch data from the service and render the contributor cards using Tailwind CSS. Reference the frontend design in contributor-widget/public/.

Usage in views:

<x-github-contributors :limit="5" />
6

Configure Laravel Scheduler

Replace the standalone cron job with Laravel's scheduler. Add to app/Console/Kernel.php: 

protected function schedule(Schedule $schedule): void
{
    // Update contributors monthly on the 1st at 2 AM UTC
    $schedule->command('github:update-contributors')
        ->monthlyOn(1, '02:00')
        ->timezone('UTC');

    // Cleanup expired cache daily at 4 AM UTC
    $schedule->command('github:cleanup-cache')
        ->dailyAt('04:00')
        ->timezone('UTC');
}

Create corresponding Artisan commands using php artisan make:command UpdateGitHubContributors

7

Integrate Laravel Cache

Use Laravel's Cache facade instead of the standalone cache service. Update the service to use: 

use Illuminate\Support\Facades\Cache;

// Memory cache (1 hour)
Cache::remember('github_contributors', 3600, function () {
    // Fetch from database or API
});

// Database cache (30 days)
Cache::driver('database')->remember('github_contributors_db', 30 * 24 * 60 * 60, function () {
    // Fetch from API
});
8

Test Integration

Run these commands to verify the integration:

# Run migrations
php artisan migrate

# Test the update command
php artisan github:update-contributors

# Verify data in database
php artisan tinker
>>> DB::table('contributors')->count();

# Clear cache and test
php artisan cache:clear
